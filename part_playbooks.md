# 操作手册 (part_playbooks)

> 每个子部分的标准操作流程 (SOP)

---

## 通用前置步骤（信息源识别 SOP）

### Step 0: 确认行政层级
- 输入: 行政区划代码或名称
- 输出: L1(省级) / L2(地级) / L3(县区级)
- 规则: 6位行政区划代码 → 前2位≠0且后4位=0→省级；前4位≠0且后2位=0→地级；全6位均有效→县区级

### Step 1: 识别出资人机构
- **动作**: 查目标地区政府门户 > "机构设置" 或 "组织机构"
- **查找**: 是否存在 "国资委" / "国资办" / "国资中心" / "国有资产管理" 字样的独立机构
- **输出**: {机构类型: S1/S2/S8, 机构名称, 机构URL}
- ⚠️ 此步仅确认机构存在与类型，**不从此页面提取任何业务数据**

### Step 2: 定位信息公开渠道
- 对**出资人机构官网**: 查找 "政务公开" / "信息公开" 一级栏目
- 对**同级财政部门**: 查找 "预决算公开" / "国有资产管理" 栏目
- 对**同级人大**(L1/L2): 查找 "监督" / "权威发布" / "报告" 栏目
- 输出: {渠道列表, 各渠道URL, 各渠道对4个Part的适配度}

### Step 3: 信息来源标注
- 所有采集的信息**必须标注**: 原始发布机构、发布日期、具体栏目、文件编号（如有）
- 转载内容必须追溯到原始发布页

## Part 1: 监管国有企业名单

### P1-SOP: 总流程

```
输入: 目标地区(行政区划代码+名称)
输出: 该地区监管企业名单(结构化) + 口径标注 + 证据链

Phase A: 信息源定位 → Phase B: 名单检索 → Phase C: 名单判定 → Phase D: 结构化抽取 → Phase E: 质检与交叉验证
```

| S8 国资中心 | "出资企业" / "管理企业" | 无统一模式 |

| 模式代码 | 描述 | 实例 | 特征 |
|---------|------|------|------|
| **LIST-A** | 专设栏目，每企业一页 | 北京国资委 `国企之窗→监管企业` (3页, 30+家) | 分页列表→点击→企业简介页（叙事文本） |

### Phase B: 名单检索

**B1. 搜索策略优先级：**

```
Step B1.1: 导航检索（最高优先级）
  → 进入出资人机构官网 → 寻找一级导航中的"企业"相关栏目
  → 匹配词: 监管企业|出资企业|企业名录|企业概况|国企之窗|省属企业|市属企业

Step B1.2: 站内搜索（次高优先级）
  → 使用网站自带搜索功能
  → 搜索词: "监管企业名单" / "出资企业名录" / "企业名录"
  → 注意: 很多政府网站站内搜索功能差，可能搜不到

Step B1.3: 搜索引擎定向检索
  → site:gzw.{region}.gov.cn "监管企业"
  → site:gzw.{region}.gov.cn "出资企业"
  → site:gzw.{region}.gov.cn "企业名录"
  → 注意: 如果是财政局 → site:czj.{region}.gov.cn "出资企业"

Step B1.4: 信息公开目录检索
  → 进入"政务公开→政府信息公开目录" → 筛选分类"机构职能"或"监管"
  → 查找"经市政府授权，监管以下企业"等表述

Step B1.5: 间接推断（最低优先级）
  → 从运行数据中提取提及的企业名称（如"XX户企业实现营收"→推断XX户名称）
  → 从年度工作报告中提取
  → 从工资分配信息公示中提取（如浙江案例）
```

### Phase C: 名单判定规则

**C1. 口径判定（6种口径等价性分析）：**

```
口径①: 监管企业名单
  定义: 国资委经本级政府授权，直接监管的国有企业列表
  范围: 仅限国资委授权监管范围，不含金融/文化/其他部门监管的国企
  特征: 通常标题含"监管企业"，发布在国资委网站
  等价性: ≈口径②（当该地区所有出资企业均归国资委时）

口径②: 所出资企业名单
  定义: 出资人机构直接出资（控股/参股）的企业列表
  范围: 强调出资关系而非监管关系
  特征: 标题含"出资企业"
  等价性: ≈口径①（国资委场景下）；≠口径①（财政局可能同时是金融类国企出资人）

口径③: 出资企业名录
  定义: 同口径②，仅为不同叫法
  等价性: ≡口径②

口径④: 国家出资企业名单
  定义: 《企业国有资产法》第5条定义——国家出资的国有独资/控股/参股企业
  范围: 法律概念，是口径①②的超集
  特征: 通常出现在法律/制度文件中，不出现在日常发布中
  等价性: ⊃口径① ⊃口径②

口径⑤: 企业名录/企业概况
  定义: 网站上的企业展示页面
  范围: 可能不完整（仅展示部分重点企业）
  特征: 多见于"国企之窗"类PR栏目
  等价性: ⊆口径①（可能是子集，需验证完整性）
  ⚠️ 高风险: 不能直接等同于完整监管名单

口径⑥: 全部国有企业统计口径
  定义: 包含所有国有独资/控股企业的统计报表口径
  范围: 远大于口径①，含所有部门监管的国企 + 各级子企业
  特征: 通常出现在国有资产管理情况报告中（如"全市共有国有企业XXX户"）
  等价性: ⊃⊃口径①（显著大于）；不可等价使用
```

**C2. 完整性判定规则：**

```
规则 JC-01: 完整名单 vs. 部分展示
  IF 页面标题含"名单"或"名录" AND 有明确的总数声明 → 高置信完整
  IF 页面标题含"概况"或"之窗" AND 无总数声明 → 可能不完整
  IF 来自"友情链接"区域 → 几乎一定不完整（仅有官网的企业才列入）
  IF 来自"通知公告"且有文号 → 高置信完整（正式发文）

规则 JC-02: 如何验证完整? 
  交叉验证法1: 比对国资委年报中的"监管XX户企业"数量与采集到的名称数量
  交叉验证法2: 比对运行数据中提及的企业名称是否都在名单中
  交叉验证法3: 比对工资分配信息公示中的企业列表

规则 JC-03: 覆盖范围确认
  IF 仅有"一级企业" → 标注: 仅一级企业，不含子企业
  IF 含二级/三级企业 → 标注: 含N级子企业
  IF 有"双管企业"独立分类 → 必须分别采集并标注分类
```

**C3. 时效性判定规则：**

```
规则 JT-01: 发布日期识别
  优先级1: 页面顶部"发布时间/发布日期/更新时间"字段
  优先级2: URL中的日期片段（如 /201812/ → 2018年12月）
  优先级3: 正文中的日期表述（如"截至2025年12月"）
  优先级4: 页面最后修改时间/网站更新记录
  ⚠️ 北京案例发现: URL日期201812（2018），但实际内容更新于2021年06月17日

规则 JT-02: 有效期判断
  IF 有明确"截至YYYY年MM月"→ 有效期=该日期
  IF 无截至日期 → 有效期=发布日期，标注"⚠️无明确截止时间"
  IF 最近2年内有企业重组/合并新闻 BUT 名单未更新 → 标注"⚠️可能过期"

规则 JT-03: 过期检测
  IF 发布日期 > 3年前 AND 期间有重组合并事件 → 标记"需更新确认"
  IF 发布日期 > 5年前 → 标记"高风险过期"
  IF 名单中某企业已被吊销/注销（工商信息验证） → 名单一定过期
```

### Phase D: 结构化抽取规则

**D1. 目标字段（Part1专用字段）：**

| 字段名 | 中文 | 类型 | 必填 | 说明 |
|--------|------|------|------|------|
| enterprise_name | 企业名称 | string | Y | 全称，含"有限公司"/"集团"等 |
| enterprise_short_name | 企业简称 | string | N | 如"首钢"、"北控" |
| enterprise_tier | 企业级次 | enum | Y | 一级企业/二级企业/三级及以下 |
| ownership_level | 归属层级 | enum | Y | 省属/市属/县属/区属 |
| supervision_type | 监管分类 | enum | Y | 监管企业/双管企业/委托监管/参股企业 |
| is_supervised | 是否为监管企业 | bool | Y | true=在监管名单中 |
| is_invested | 是否为所出资企业 | bool | Y | true=出资人直接出资 |
| caliber | 口径标注 | enum | Y | 口径①~⑥之一 |
| list_completeness | 名单完整性 | enum | Y | 完整/部分/推断 |
| source_url | 来源链接 | string | Y | 具体页面URL |
| source_type_code | 信息源类型 | enum | Y | S1~S8 |
| evidence_snippet | 证据片段 | string | Y | 原文引用（≤200字） |
| publish_date | 发布日期 | date | Y | YYYY-MM-DD |
| effective_date | 数据截至日期 | date | N | 若原文有"截至"则填 |
| page_pattern | 页面模式 | enum | Y | LIST-A~LIST-F |
| extraction_method | 抽取方法 | enum | Y | 导航/搜索/推断/附件 |

**D2. 抽取规则（按页面模式分化）：**

```
模式 LIST-A (每企业一页):
  Step 1: 从列表页提取所有企业链接文本 → enterprise_name
  Step 2: 分页处理（注意翻页）→ 获取完整列表
  Step 3: 如有分类标签（监管企业/双管企业）→ supervision_type
  Step 4: 进入详情页提取企业简介（可选）
  注意: 确认列表是否有分页（北京案例: 共3页）

模式 LIST-B (散见于友情链接/新闻):
  Step 1: 抓取footer区域"省属企业"/"市属企业"下拉列表 → enterprise_name
  Step 2: 从运行数据文章中提取提及的企业名称 → 补充列表
  Step 3: 从工资分配信息公示中提取企业名称 → 交叉验证
  Step 4: 标注 list_completeness = "部分"
  ⚠️ 此模式下名单大概率不完整，必须交叉验证

模式 LIST-C (单页清单):
  Step 1: 抽取表格/文本中的所有企业名称
  Step 2: 如有序号 → 验证总数 = 最大序号
  Step 3: 如有分类（按行业/按功能类型）→ 保留分类信息

模式 LIST-D (附件PDF/Word):
  Step 1: 定位并下载附件
  Step 2: 解析PDF/Word中的表格/清单
  Step 3: 提取全部企业名称和附属字段
  ⚠️ PDF表格解析可能出错，需人工抽检

模式 LIST-E (仅有数量):
  Step 1: 提取"监管XX户企业"的数量 → 记录
  Step 2: 标注 list_completeness = "仅数量"
  Step 3: 触发回退流程（Phase E）

模式 LIST-F (双分类):
  Step 1: 分别抽取"监管企业"和"双管企业"两个列表
  Step 2: 为每个列表分别标注 supervision_type
  Step 3: 合并输出时标注分类
```

### Phase E: 失败回退与交叉验证

**E1. 名单获取失败回退链：**

```
回退链 FB-01: 国资委无名单页
  Step 1: 检查国资委网站"政府信息公开指南"→ 其中通常说明"本委监管企业XX户"
  Step 2: 检查国资委"通知公告"→ 搜索"企业名录""企业名单"
  Step 3: 检查国资委首页footer"友情链接"或"省属企业"链接区
  Step 4: 回退到 S2 财政局 → "国有资产管理"栏目
  Step 5: 回退到 S3 政府门户 → "国有企业"或"国有经济"栏目
  Step 6: 回退到国务院国资委地方扫描（仅作线索发现）

回退链 FB-02: 仅获取到部分名单
  Step 1: 比对已获取数量与官方声明的总数
  Step 2: 搜索"工资分配信息公示" → 通常含完整企业列表
  Step 3: 搜索"年度运行数据" → 可能附带企业清单
  Step 4: 搜索"负责人经营业绩考核结果" → 含企业列表
  Step 5: 搜索"企业负责人薪酬" → 含企业列表
  Step 6: 若仍不完整 → 标注缺口 + 建议依申请公开

回退链 FB-03: 只有图表/PDF/扫描件
  Step 1: 如果是可选择文本的PDF → 直接解析
  Step 2: 如果是扫描件PDF → OCR处理
  Step 3: 如果是图片 → OCR处理
  Step 4: OCR结果与上下文交叉验证（企业名称应能在工商系统验证）
  
回退链 FB-04: 名单分散在多篇文章中
  Step 1: 收集所有提及"监管企业""出资企业"的文章
  Step 2: 提取每篇中出现的企业名称
  Step 3: 去重合并
  Step 4: 标注 extraction_method = "推断"，list_completeness = "推断"
  Step 5: 交叉验证: 与运行数据/薪酬公示中的企业名称比对

回退链 FB-05: 县区级无任何信息源
  Step 1: 查上级（地市级）国资委是否有该县区的信息
  Step 2: 查县区政府门户"机构设置"确认出资人
  Step 3: 查县区财政局"国有资产管理"栏目
  Step 4: 查县区国有平台公司官网（城投/城建/国投等）
  Step 5: 若全无 → 标注"该县区无公开监管企业名单"
```

**E2. 质检检查清单（Part1专用）：**

```
□ QC-P1-01: 口径是否已标注? (口径①~⑥中的哪一个)
□ QC-P1-02: 完整性是否已判定? (完整/部分/推断/仅数量)
□ QC-P1-03: 是否区分了"监管企业"和"双管企业"?
□ QC-P1-04: 企业数量是否与官方声明一致?
□ QC-P1-05: 发布日期和有效期是否已标注?
□ QC-P1-06: 是否有过期风险?（与最新重组新闻比对）
□ QC-P1-07: 证据链是否完整?（source_url可访问+evidence_snippet有原文）
□ QC-P1-08: 是否排除了央企名录的误混入?
□ QC-P1-09: 金融类/文化类国企是否已标注不在本名单中?
□ QC-P1-10: 如果是LIST-B/LIST-E模式，是否已触发交叉验证?
```

### Phase C: 期间与口径判定

**C1. 口径判定（4种口径 + 等价性分析）：**

```
口径P2-①: 省属/市属/县属监管企业运行
  定义: 同级国资委直接监管的企业集团的汇总运行数据
  来源: 国资委自行统计或财务快报汇总
  特征: 标题含"省属企业""市属企业"
  适用场景: 了解本级国资委监管范围内的企业经营状况
  等价关系: ≈口径P2-③（当出资人=国资委时）

口径P2-②: 本地区国有及国有控股企业运行
  定义: 覆盖本地区全部非金融类国有及国有控股企业
  来源: 财政部门统计（全口径）
  特征: 标题含"国有及国有控股企业""全国/全省/全市"
  适用场景: 了解全口径国有资本运营状况
  等价关系: ⊃口径P2-①（范围更大，含非监管国企）
  ⚠️ 全国口径数据由财政部编制，SASAC网站仅转载

口径P2-③: 国资委监管企业运行
  定义: 同口径P2-①的另一种表述
  等价关系: ≡口径P2-①

口径P2-④: 国企改革运行情况
  定义: 侧重改革进展+效果评估，指标可能包含混改数量、资产证券化率等
  来源: 国资委改革专栏/工作报告
  特征: 标题含"国企改革""改革进展""改革成效"
  等价关系: ⊄口径P2-①（不同维度，不可直接比较运行数据）
  ⚠️ 这不是标准运行数据，含定性内容

口径等价性速查:
  P2-①省属企业 ≡ P2-③国资委监管企业   (同一口径不同名称)
  P2-②全口径 ⊃ P2-①省属             (全口径包含监管口径)
  P2-④改革情况 ⊄ P2-①②③             (不同维度，不直接等价)
  
  ⚠️ P2-①和P2-②的数据绝不能混用！如果混用将导致:
  - 营业收入: 全口径>>监管口径（因含非监管国企）
  - 资产规模: 全口径>>监管口径
  - 利润: 可能方向不同（监管企业盈利+非监管企业亏损时）
```

**C2. 期间识别规则：**

```
规则 PT-01: 标题级期间识别（最高优先级）
  模式1: "{year}年1-{N}月" → 当年累计截至第N月
  模式2: "{year}年{N}月" / "{year}年{N}月份" → 单月数据
  模式3: "{year}年全年" / "{year}年1-12月" → 年度数据
  模式4: "{year}年一季度" → 1-3月
  模式5: "{year}年上半年" → 1-6月
  模式6: "{year}年前三季度" → 1-9月

规则 PT-02: 正文级期间识别（补充标题不明确的情况）
  模式A: "截至{N}月末" → 时点数据（资产/负债用）
  模式B: "1—{N}月份" / "1-{N}月" → 累计数据（收入/利润用）
  模式C: "{N}月份" → 单月数据
  模式D: "前{N}个月" → 累计数据
  ⚠️ 浙江案例发现: 同一篇文章内同时包含"1月份"和"1-2月份"两组数据

规则 PT-03: 累计 vs 单月判定
  IF 标题含"1-{N}月" → 累计数据
  IF 标题含"{N}月份" → 单月数据
  IF 正文同时有累计和单月 → 分别抽取，标注period_type
  ⚠️ 绝大多数国资运行数据为"当年累计"（1-N月），而非单月。单月数据少见。

规则 PT-04: 数据时滞判定
  全国口径(财政部): 通常滞后约30天（如1-2月数据→4月1日发布）
  省级口径: 通常滞后15~30天（浙江: 1-2月→3月18日）
  地市级口径: 滞后不定，30~90天均有可能
  ⚠️ 发布日期 ≠ 数据期间截止日期，必须分别标注
```

### Phase D: 指标抽取

**D1. 指标分层体系：**

```
第一层: 核心指标（几乎所有运行数据都会包含）
  ├─ 营业总收入 / 营业收入
  ├─ 利润总额
  ├─ 资产总额（时点数据，"截至X月末"）
  └─ 资产负债率（时点数据）

第二层: 标准指标（全国月报包含，地方可能包含）
  ├─ 应交税费 / 已交税费
  ├─ 负债总额
  └─ 成本费用总额（较少公开）

第三层: 扩展指标（部分地方公开，非标准化）
  ├─ 工业总产值
  ├─ 上缴税费 / 实际交税
  ├─ 固定资产投资 / 完成投资
  ├─ 研发投入 / 研发投入强度
  ├─ 净资产收益率
  ├─ 经济增加值(EVA)
  ├─ 全员劳动生产率
  └─ 基础设施投资额

第四层: 地方自定义指标（甘肃案例等）
  ├─ 战略性新兴产业营业收入
  ├─ "三新"经济营业收入
  ├─ "高端化、智能化、绿色化"改造投资
  └─ 其他省/市自行定义的特色指标
```

**D2. 指标抽取规则（按内容格式分化）：**

```
格式 RUN-A（结构化正文，如财政部月报/浙江模式）:
  Step 1: 识别指标名称（匹配指标词库中的词组）
  Step 2: 提取紧邻的数值（格式: X,XXX.X亿元 或 XX.X%）
  Step 3: 提取同比变化（"同比增长/下降XX.X%"或"同比上升/下降X.X个百分点"）
  Step 4: 识别单位（亿元/万亿元/百分比/个百分点）
  Step 5: 区分流量指标（1-N月累计）和时点指标（N月末）

  浙江案例解析:
  "据财务快报统计，1-2月份省属企业实现营业收入2,399.1亿元，同比增长2.2%"
  → indicator=营业收入, value=2399.1, unit=亿元, yoy_change=+2.2%, period=1-2月

格式 RUN-B（新闻叙事格式，如甘肃模式）:
  Step 1: 全文扫描，按句/段落分割
  Step 2: 对每句话搜索指标名称关键词
  Step 3: 提取数值——注意: 部分指标可能只有同比没有绝对值!
  Step 4: 标注哪些指标有绝对值、哪些仅有同比
  
  甘肃案例解析:
  "截至2月末...资产总额达19821.14亿元，同比增长6.97%"  → 有绝对值+同比
  "工业总产值同比增长32.94%"  → 仅有同比，无绝对值
  "营业总收入同比增长14.72%"  → 仅有同比，无绝对值
  "研发投入16.68亿元"  → 有绝对值，同比在后文（11.9%是改造投资的同比非研发的）

格式 RUN-C（表格格式，PDF/Excel附件）:
  Step 1: 定位表格
  Step 2: 识别表头（指标名称行 + 期间列）
  Step 3: 按行抽取指标名称+数值+同比
  Step 4: 注意: 表格可能含多个口径（竖向分区）
```

**D3. 数值解析规则：**

```
规则 NV-01: 数值格式标准化
  "2,399.1亿元" → value=2399.1, unit="亿元"
  "65.4%" → value=65.4, unit="%"
  "0.5个百分点" → value=0.5, unit="个百分点"
  "约2400亿" → value=2400, unit="亿元", is_approximate=true

规则 NV-02: 同比方向识别
  "同比增长X%" → yoy_direction="+", yoy_pct=X
  "同比下降X%" → yoy_direction="-", yoy_pct=X
  "同比上升X个百分点" → yoy_direction="+", yoy_bp=X (百分比指标专用)
  "同比下降X个百分点" → yoy_direction="-", yoy_bp=X
  "同比持平" → yoy_direction="0"
  "同比回落X个百分点" → yoy_direction="-", yoy_bp=X
  "剔除政策性因素后同比增长X%" → yoy_adjusted=true, yoy_pct=X

规则 NV-03: 数值验证
  IF 资产负债率>100% → 数据异常，标记
  IF 营收同比>300% → 可能有重组并入因素，标记
  IF 利润为负且"同比增长" → 减亏（亏损额减小=增长），标记特殊
```

### Phase E: 真实性与时效性判定

**E1. 原始发布判定：**

```
规则 AU-01: 来源标注检测
  IF "文章来源：XX网站" AND 来源≠当前网站 → 转载
  全国口径案例: SASAC网页标注"文章来源：财政部网站" → 原始发布者=财政部
  
规则 AU-02: 发布机构层级判定
  IF 地方运行数据出现在国务院国资委"地方"栏目 → 二手来源
  → 应追溯到具体省级国资委原始发布页
  
规则 AU-03: "财务快报" vs "统计数据"
  IF 正文含"据财务快报统计" → 数据来源为企业财务快报（初步数据）
  IF 正文含"据统计" / 无特别说明 → 可能为正式统计数据
  ⚠️ 财务快报数据可能在正式年报/决算后有调整
```

**E2. 同期多版本处理：**

```
规则 VER-01: 同月多篇版本
  IF 同一期间存在多篇文章 → 取发布日期最晚的
  IF 有"更正""修订""调整后"字样 → 取更正版
  IF 国资委版本和政府门户版本数据不一致 → 以原始发布方为准

规则 VER-02: 旧文改版检测
  IF URL日期远早于内容描述的期间 → 检查是否老页面更新内容
  IF 同一URL的内容在不同时点采集结果不同 → 标注"页面已更新"

规则 VER-03: 数据修正链
  IF 后期月份的累计数据与前期月份不连贯 → 可能有追溯调整
  例: 1-6月利润100亿，而此前1-5月利润95亿 → 6月利润仅5亿（合理）
  例: 1-6月利润100亿，而此前1-5月利润110亿 → 可能有追溯重述（标记）
```

### Phase F: 失败回退与质检

**F1. 运行数据获取失败回退链：**

```
回退链 FB-P2-01: 国资委无月报
  Step 1: 检查国资委"通知公告"或"工作动态"中的年度工作报告
  Step 2: 检查政府门户"政府工作报告"中的国资/国企段落
  Step 3: 检查"政府信息公开年报"中可能包含的统计数据
  Step 4: 降级为年度工作总结中提取的有限数据点
  Step 5: 标注 data_granularity = "年度"，fallback_used = true

回退链 FB-P2-02: 只有年报/年度总结
  Step 1: 从年度工作总结中提取年度累计指标
  Step 2: 从人大"国有资产管理情况报告"中提取运行数据段落
  Step 3: 从审计报告中提取相关数据
  Step 4: 标注可用的最细粒度

回退链 FB-P2-03: 只有新闻稿/通稿
  Step 1: 从新闻稿中提取所有可识别的指标数值（RUN-B格式解析）
  Step 2: 标注 data_type = "新闻引用"，completeness = "部分"
  Step 3: 交叉验证: 与全国口径数据的区域趋势对比

回退链 FB-P2-04: 只有图片/不可复制PDF
  Step 1: OCR处理提取表格数据
  Step 2: 人工核对关键数值
  Step 3: 标注 extraction_method = "OCR"

回退链 FB-P2-05: 地市级/县区级无运行数据
  Step 1: 查上级国资委是否有分地区数据
  Step 2: 查地方统计局"统计年鉴"中的国有经济章节
  Step 3: 查地方政府工作报告中的相关段落
  Step 4: 查国有企业债券募集说明书/年报（如有发债/上市企业）
  Step 5: 标注"该地区无公开运行数据"

回退链 FB-P2-06: 某一期间缺失（如缺1月/2月单报）
  Step 1: IF 有1-2月累计 → 用累计数据，不强求单月
  Step 2: IF 有上一期累计+本期累计 → 差值推算单月（标注"推算"）
  Step 3: IF 为1月(开年) → 很多地方不单独发1月，正常直接用1-2月
  ⚠️ 关键规律: 1月通常不单独发布，最早发布为1-2月合并
```

### P4B: 国有资产管理情况报告

#### Phase A: 信息源确定

**发布机制说明**:

2018年起, 国务院每年向全国人大常委会报告国有资产管理情况:
- **综合报告**: 每年固定提交(书面), 覆盖四类国有资产全口径

  - 企业国有资产(不含金融企业) → 2018, 2022, ...

  - 行政事业性国有资产 → 2020, 2024年轮次但实际为2024企业, ...
  - 国有自然资源(资产) → 2021, 2025, ...
  
**⚠️ 注意**: 专项报告轮次并非严格固定,实际年份需以当年常委会议程为准

**地方**: 省/市/县人大参照全国制度建立本级国有资产报告制度, 但进度不一

**发布主体优先级** (高→低):

#### Phase C: 判定规则

**C4B.1 区分"综合报告"与"专项报告"**:

| 特征 | 综合报告 | 专项报告 |
|------|---------|---------|
| 覆盖范围 | 四类全口径(企业+金融+行政事业+自然资源) | 单一类别深入 |
| 数据深度 | 每类1~2个核心指标(资产总额+负债/权益) | 包含10+个指标, 含问题和建议 |
| 提交方式 | 书面报告(不作口头说明) | 受国务院委托口头报告 |
| 篇幅 | 5000~8000字 | 10000~15000字 |
| 标题 | "XX年度国有资产管理情况综合报告" | "XX年度XX国有资产(不含…)管理情况专项报告" |

**C4B.2 区分"本级"与"全辖区"**:
- 省级综合报告通常覆盖"全省"(汇总), 但核心数据区分"省属"和"全省"
- 需提取时标注: scope = "provincial_soe" / "all_province"
- "地方国有企业" 在中央报告中 = 除央企外的全部

**C4B.3 判断是否覆盖企业国有资产**:
- 综合报告: 必然覆盖(第一/第二章)
- 企业专项报告: 完全覆盖(但"不含金融企业")
- 金融专项、行政事业专项、自然资源专项: 不覆盖企业国有资产
- **规则**: report_covers_enterprise = (report_type == "comprehensive") OR (special_type == "enterprise")

**C4B.4 区分报告版本(极高误判风险)**:

| 版本 | 来源 | 法律效力 | 完整度 |
|------|------|---------|--------|
| 受国务院委托报告(全文) | 政府公报/财政部门 | 最高 | 最完整 |
| 人大网站发布版 | 人大门户 | 高(经审议) | 可能有删节 |
| 审议意见/意见和建议 | 人大门户→审议意见栏 | 审议产出(非报告本身) | 仅含委员意见 |
| 新闻摘要 | 新华社/人大新闻 | 低(媒体加工) | 高度精简 |
| 审计交叉引用 | 审计厅 | 补充(非原始) | 部分数据 |

**规则**: 必须标注 doc_version, 优先级: 全文 > 人大发布版 > 审议意见 > 新闻

#### Phase E: 失败回退

**FB-P4B-01: 人大门户无报告全文(仅有审议意见)** →
1. 搜索政府门户(gov.cn/公报)
2. 搜索财政厅/财政局"国有资产管理"栏目
3. 搜索国资委网站转载
4. 从审议意见中提取被引用的数据(标注 source = "npc_review_opinion")

**FB-P4B-02: 省/市/县无国有资产报告制度** →
1. 搜索该级人大常委会会议记录
2. 搜索政府工作报告中"国有资产"相关段落
3. 搜索国资委年度工作总结
4. 标注 has_reporting_system = false

**FB-P4B-03: 报告只有摘要无全文** →
1. 标注 completeness = "summary"
2. 搜索政府公报/政府门户→信息公开→报告
3. 搜索"关于落实…审议意见的报告"(可能转载原文数据)
4. 记录摘要中的可用数据(通常包含2~3个核心指标)

**FB-P4B-04: 综合报告存在但企业部分数据不足** →
1. 查该年度是否有企业专项报告(4年轮换)
2. 如非企业专项报告年份 → 仅综合报告数据可用
3. 搜索国资委年报/报告补充企业数据
4. 标注 enterprise_data_source = "comprehensive_report_only"

**FB-P4B-05: 历史年份报告缺失** →
1. 2018年以前无此制度, 不搜索
2. 2018~2020年部分地方制度未建立, 属正常缺失
3. 搜索政府公报/地方人大公报
