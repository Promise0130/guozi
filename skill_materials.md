# Skill 可复用材料 (skill_materials)

> 最终沉淀的、可直接写入 skill 文件的规则、模板和判断逻辑

## 1. 关键概念定义

| 概念 | 定义 | 易混淆点 |
|------|------|----------|
| 履行出资人职责机构 | 代表本级政府对国有企业行使出资人权利的机构 | 不一定是国资委；可以是财政局、国资中心、政府授权机构 |
| 监管企业 | 出资人机构直接监管的一级企业集团（含其全资/控股子企业） | ≠ 全部国有企业；仅为授权监管范围内企业 |
| 国有资本经营预算 | "四本预算"之一，专门反映国有资本收益收缴和支出安排 | 编制主体是财政部门，不是国资委 |
| 国有资产管理情况报告 | 政府向同级人大常委会提交的专项报告（含4类国有资产） | 人大审议版 vs. 政府发布版可能存在差异 |
| 4类国有资产 | ①企业经营性 ②金融类 ③行政事业性 ④国有自然资源 | 国资委仅管第①类的一部分 |
| 双管企业 | 事业单位转企后由国资委监管的企业 | 北京等地有独立的"双管企业"分类 |

## 2. 判断规则库

### 规则组 R1: 出资人机构识别

```
R1.1 省级行政区 → 默认存在独立国资委
  例外: 无（2026年全部31个省级行政区均有独立国资委）

R1.2 地级行政区 → 先查政府门户"机构设置"确认
  ├─ 存在"国资委" → 使用国资委渠道
  ├─ 存在"国资办/国资中心" → 使用该机构渠道
  ├─ 仅"财政局" → 财政局国资股为出资人
  └─ 不确定 → 查"三定方案"或政府公报

R1.3 县区级行政区 → 高概率无独立国资委
  ├─ 直辖市的区 → 通常有区国资委（验证:北京各区有）
  ├─ 省会/副省级城市的区 → 可能有区国资局/国资办
  └─ 普通地级市辖区/县 → 大概率由财政局代管
```

### 规则组 R2: 信息源优先级

```
R2.1 Part1(监管企业名单):
  优先级: S1国资委 > S8国资中心 > S2财政局 > S3政府门户
  禁止: 从S3"机构设置"页取企业名单数据

R2.2 Part2(月度/年度运行):
  优先级: S1国资委 > S2财政厅(全口径) > S6企业官网 > S7审计
  注意: 国资委口径=监管企业；财政口径=国有资本经营预算覆盖范围

R2.3 Part3(重组/基金/主责主业):
  优先级: S1国资委 > S3政府门户(批复文件) > S6企业官网 > C类第三方
  注意: 区分政府主导的战略重组与市场化的产权转让

R2.4 Part4(国资经营预算/资产报告):
  预算类优先级: S2财政 > S5预决算平台 > S4人大
  报告类优先级: S4人大 > S3政府门户 > S1国资委
  注意: 预算的编制阶段(草案)、执行阶段(月/季报)、决算阶段需分别处理
```

### 规则组 R3: 内容真伪判断

```
R3.1 转载检测:
  IF 文章标注"来源: XXX" AND 来源≠当前网站 → 标记为转载，追溯原始来源
  IF 文章URL域名≠当前查阅网站域名 → 外链，非原始发布

R3.2 正文完整性:
  IF 页面仅含标题列表/目录索引 → 未获取正文，需打开附件
  IF 预算数据页无PDF/Excel附件链接 → 可能是目录页，非正文

R3.3 版本区分:
  IF 来自人大网站 → 标记"人大审议版"
  IF 来自财政部门网站 → 标记"财政部门发布版"
  IF 来自政府门户 → 需判断是政府提交版还是转载
```

## 7. Part1 专项材料: 监管企业名单

### 7.1 名单口径判断规则 (规则组 R4)

```
R4.1 口径识别流程:
  Step 1: 检查页面标题/栏目名称 → 匹配到"监管企业"→口径①
  Step 2: 检查页面标题 → 匹配到"出资企业" → 口径②
  Step 3: 检查正文 → 提到"经XX政府授权监管" → 口径①
  Step 4: 检查正文 → 提到"直接出资/控股" → 口径②
  Step 5: 检查文件编号 → 来自国资委 → 口径①或②; 来自财政 → 可能口径④
  Step 6: 检查数据总量 → 若>>同地区国资委公开数 → 可能是口径④或⑥

R4.2 口径等价性速查:
  口径①监管企业 ≈ 口径②出资企业    (当且仅当出资人=国资委时)
  口径②出资企业 ≡ 口径③出资企业名录  (完全等价，仅名称差异)
  口径④国家出资企业 ⊃ 口径①          (法律超集)
  口径⑤企业概况 ⊆ 口径① ⚠️          (可能是子集，高风险不完整)
  口径⑥统计口径 ⊃⊃ 口径①             (数量级差异, 绝不等价)

R4.3 口径冲突处理:
  IF 同一地区从不同信息源获取到不同数量 → 标注两个口径，不合并
  IF 标题说"监管企业"但数量远超预期 → 检查是否包含子企业(二级/三级)
  IF 标题说"国有企业"但数量远少于预期 → 检查是否仅含一级企业
```

### 7.2 名单抽取模板 (模板 T2)

```
模板 T2: 监管企业名单抽取流程

输入: 目标地区、已识别的出资人机构及URL
输出: 结构化名单 + 口径标注 + 完整性评估

Step 1. 定位名单页 (Phase A+B)
  1a. 进入出资人机构官网
  1b. 按优先级检索: 导航→站内搜索→搜索引擎→信息公开目录→间接推断
  1c. 输出: {名单URL, page_pattern(LIST-A~F)}

Step 2. 判定口径 (Phase C - R4规则)
  2a. 识别口径编号(①~⑥)
  2b. 检查是否有双分类(监管+双管)
  2c. 输出: {caliber, has_dual_category}

Step 3. 抽取名单 (Phase D - 按page_pattern分支)
  3a. 按对应模式执行抽取
  3b. 处理分页（LIST-A/LIST-F必须翻页）
  3c. 填充字段: enterprise_name, enterprise_tier, supervision_type等
  3d. 输出: {企业列表[], total_count_extracted}

Step 4. 判定完整性 (Phase C - JC规则)
  4a. 比对total_count_extracted与total_count_claimed
  4b. 执行交叉验证（如需要）
  4c. 输出: {list_completeness, 缺口说明}

Step 5. 标注时效性 (Phase C - JT规则)
  5a. 识别发布日期和有效期
  5b. 检查过期风险
  5c. 输出: {publish_date, effective_date, 过期风险}

Step 6. 质检 (Phase E - QC清单)
  6a. 逐项检查QC-P1-01~10
  6b. 输出: {通过/未通过项列表}

Step 7. 失败处理 (Phase E - FB链)
  7a. 如Step 1~3任一步失败 → 触发对应回退链
  7b. 标注回退路径
  7c. 输出: {fallback_chain_used, fallback_result}
```

## 8. Part2 专项材料: 月度/年度运行

### 8.1 运行数据口径判断规则 (规则组 R5)

```
R5.1 口径识别流程:
  Step 1: 检查标题→含"省属企业"/"市属企业"→P2-①(监管口径)
  Step 2: 检查标题→含"全国/全省/全市"+"国有及国有控股"→P2-②(全口径)
  Step 3: 检查标题→含"国资委监管"→P2-③(≡P2-①)
  Step 4: 检查标题→含"国企改革"/"改革进展"→P2-④(改革维度)
  Step 5: 检查文章来源→"财政部网站"→大概率P2-②
  Step 6: 检查注脚→"不含金融企业"→确认为P2-②全口径(财政部定义)

R5.2 口径等价性速查:
  P2-① ≡ P2-③  (同一口径不同表述)
  P2-② ⊃ P2-①  (全口径包含监管口径,不可简单比较)
  P2-④ ⊄ P2-①②③ (不同维度,不直接等价)
  
R5.3 口径混用禁令:
  ⛔ 同一表格内不得混合"省属企业"和"全口径"数据
  ⛔ 不得用全口径营收减去省属营收来推算"非监管国企"营收(统计范围和方法不同)
  ⛔ 不得将P2-④改革情况中的指标(如混改数量)与P2-①②的财务指标放同一分析框架
```

### 8.2 期间识别规则 (规则组 R6)

```
R6.1 标题级识别（正则模式）:
  /(\d{4})年1[-—至](\d{1,2})月/      → 累计, year=$1, end_month=$2
  /(\d{4})年(\d{1,2})月份?/            → 单月, year=$1, month=$2
  /(\d{4})年全年/                       → 年度, year=$1
  /(\d{4})年一季度/                     → 累计1-3月
  /(\d{4})年上半年/                     → 累计1-6月
  /(\d{4})年前三季度/                   → 累计1-9月
  /(\d{4})年1[-—至]12月/               → 年度

R6.2 正文级识别:
  /截至(\d{1,2})月末/                   → 时点数据(资产/负债)
  /1[—-](\d{1,2})月份?/                → 累计
  /前(\d{1,2})个月/                     → 累计
  /(\d{1,2})月当月/                     → 单月(明确)
  
R6.3 消歧规则:
  IF 同一文章含"1月份...X亿元"和"1-2月份...Y亿元" → 分两条记录
  IF 标题说"1-2月"但正文段落只说"2月末" → 该段落是时点(资产类指标)
  IF 标题无月份但正文有 → 以正文为准

R6.4 1月特殊规则:
  大多数地方不单独发布1月数据，最早为"1-2月"合并发布
  IF 检索"1月经营情况"无结果 → 不标记为缺失，直接查"1-2月"
  IF 某省确有1月单月 → 同时也会有1-2月累计
```

### 8.3 指标标准化规则 (规则组 R7)

```
R7.1 名称标准化:
  "营业收入" → 标准名="营业总收入" (注: 浙江省用"营业收入",财政部用"营业总收入")
  "利润" (无修饰) → 需确认是"利润总额"还是"净利润",默认映射"利润总额"
  "主营业务收入" → 标准名="营业总收入" (旧会计准则名称)
  "净利润" → 不映射为利润总额,保持为独立指标

R7.2 单位标准化:
  输出统一为: 亿元(金额类), %(百分比类), 个百分点(变化类), 万元/人(效率类)
  IF 原文为"万亿元" → 转换为亿元(×10000)
  IF 原文为"万元" → 转换为亿元(÷10000)
  IF 原文为"百分点" → 保留为"个百分点"

R7.3 流量vs时点判定:
  流量指标(期间累计): 营业总收入、利润总额、应交税费、成本费用、投资额、研发投入
  时点指标(期末余额): 资产总额、负债总额、资产负债率
  ⚠️ 流量指标期间="1-N月", 时点指标期间="截至N月末"
  ⚠️ 不得将时点指标理解为期间累计
  
R7.4 同比特殊处理:
  "同比增长X%" → yoy_direction="+", yoy_pct=X
  "同比上升X个百分点" → yoy_direction="+", yoy_bp=X (仅限百分比型指标)
  "剔除政策性因素后同比增长X%" → yoy_adjusted=true
  "同比减亏" / "亏损额同比减少X%" → 利润为负但趋势改善, yoy_direction="+"
```

## §10 Part4 材料: 国有资本经营预算 / 国有资产管理情况报告

### §10.1 四本预算体系定位规则 R8

**R8.1 四本预算识别** (防止混淆的基础知识):
1. **一般公共预算** — 税收+非税收入, 覆盖教育/医疗/社保等公共支出
2. **政府性基金预算** — 专项收入(土地出让金等), 专款专用
3. **国有资本经营预算** — 国企利润上缴, 用于国企自身(资本金注入/历史遗留/公益补贴) + 调入一般公共预算
4. **社会保险基金预算** — 社保缴费, 封闭运行

**R8.2 唯一性判定**: 
- 文件必须明确提及"国有资本经营预算"这6个字
- "一般公共预算"中涉及国企补贴的不是国资预算
- "政府性基金预算"中的土地出让不是国资预算
- "社会保险基金预算"永远不相关
- "部门预算"是微观层面(单个部门), 不是宏观的四本预算

**R8.3 收支平衡公式**:
- 收入总量 = 本年收入 + 上年结转
- 支出总量 = 本年支出 + 调出资金(调入一般公共预算)
- 收入总量 = 支出总量 (平衡)
- 验证: total_revenue + carryover_in = total_expenditure + transfer_out + new_carryover

### §10.2 预算口径判定规则 R9

**R9.1 本级 vs 全辖区**:
| 标志 | 判定 | 例子 |
|------|------|------|
| 标题含"本级" | scope = own_level | "2026年本级国有资本经营预算" |
| 标题含"全省/全市" | scope = consolidated | "2026年全省国有资本经营预算" |
| 预算报告中"中央" | scope = central | 预算报告§(六) |
| 预算报告中"地方" | scope = local_aggregated | 预算报告§(六)的地方段 |
| 预算报告中"全国" | scope = national_total | 汇总中央和地方 |
| 无明确标注 | 默认own_level + uncertainty | 需交叉验证 |

**R9.2 预算 vs 决算**:
- 预算(draft/approved): 年初计划数
- 决算(settlement): 年末实际执行数
- 预算执行(execution): 中间进度数
- **同一年度优先级**: settlement > execution > approved > draft
- **时效**: 决算通常滞后6~9个月(如2025年决算在2026年7~9月公开)

**R9.3 是否含金融企业**:
- 中央层级: 国资经营预算"说明"文件中明确标注"不含金融"
- 但预算收入表可能包含"金融企业利润收入"(少量)和"金融企业股息红利收入"
- **规则**: 检查实际表内是否有金融行项; 检查"说明"中编制范围

### §10.3 国有资产报告制度规则 R10

### §10.4 预算抽取模板 T5

**T5 国有资本经营预算抽取工作流**:

Step 1: **定位信息源**
- 首选: 财政厅/预算公开平台 → 搜索"国有资本经营预算"
- 次选: 人大门户 → 预算草案报告全文
- 再次: 政府门户 → 预算公开专栏

Step 2: **确定文件类型**
- 有独立预算表(BUD-T1/T2)? → 优先抽取表数据
- 有预算说明(BUD-T3)? → 提取编制范围+收取比例

Step 3: **判定口径**
- 本级 vs 全辖区 (R9.1)
- 预算 vs 决算 (R9.2)
- 是否含金融 (R9.3)

Step 4: **抽取核心指标**
- 收入: total_revenue + revenue_by_category(利润/股息/产权/清算/其他)
- 支出: total_expenditure + expenditure_by_category(历史/注资/补贴/转移)
- 关键: transfer_out(调入一般公共预算)
- 附加: carryover, yoy变化, 利润收取比例

Step 5: **验证平衡**
- 收入总量 = 支出总量(R8.3)
- 若不平衡: 检查是否遗漏结转或调出

Step 6: **质量检查**
- [ ] 文件标题确含"国有资本经营预算"?

### §11.2 通用真实性判定规则 (UR-02)

**归并来源**: R3(真伪判断) + AU(真实性) + VER(版本) + 各Part Phase E

```
UR-02.1 原始来源判定（全Part通用）:
  Step 1: 检查"来源"/"文章来源"标注
    IF 标注机构 ≠ 当前网站所属机构 → is_repost = true
    IF 无标注 → 默认为原始发布, 但 confidence -= 0.05
  Step 2: 检查URL域名
    IF URL域名 ≠ 预期发布机构域名 → 确认为转载
  Step 3: 追溯原始
    IF is_repost = true → 记录 original_publisher, 尝试获取原始URL
    特殊: 全国口径月报 → 原始发布者=财政部(SASAC转载)
    特殊: 省级SASAC"地方"栏目 → 原始发布者=地市级SASAC

UR-02.2 版本层次判定（全Part通用, 5级）:
  LV-1 原始正式发布 → confidence_base = 0.95
    标志: 机构官网+发文字号+正式栏目 | 预算表/决算表 | 正式公告
  LV-2 人大/政府审议/发布版 → confidence_base = 0.90
    标志: 人大网站+政府公报+正式报告页 | "权威发布"栏目
  LV-3 新闻通稿 → confidence_base = 0.75
    标志: 政府新闻办/企业新闻中心/SASAC新闻发布
  LV-4 审议意见/转载摘要 → confidence_base = 0.60
    标志: "意见和建议"/"审议发言"/"摘要"/"节选"
    ⚠️ FM-P4B-01: 审议意见≠报告全文
  LV-5 媒体二手报道 → confidence_base = 0.40
    标志: 财经媒体+"据悉"/"消息称"/无出处

UR-02.3 多版本选取规则（全Part通用）:
  规则1: 同一内容多个版本 → 取版本层次最高的(LV-1 > LV-5)
  规则2: 同一版本层次多篇 → 取发布日期最晚的
  规则3: 有"更正""修订""调整后"字样 → 取更正版
  规则4: 如不同来源数据矛盾 → 不强行选择, 保留所有版本+标注差异
  规则5: 标注所选版本的 doc_version + 标注未选版本的 alt_versions[]

UR-02.4 正文完整性判定（全Part通用）:
  IF 页面仅含标题列表/目录索引 → completeness = "目录页", 需打开附件
  IF 页面标题含"(摘要)" → completeness = "摘要", 优先找全文版
  IF PDF无法复制文字(扫描件) → completeness = "需OCR"
  IF 表格仅为图片 → completeness = "需OCR"
```

```

### §11.5 通用命中规则框架 (UR-05)

**归并来源**: 各Part的TITLE_HIT + BODY_HIT + 排除词

```
UR-05.1 三级命中架构（全Part通用）:

  TITLE_HIT (标题级, 权重最高):

### §11.7 通用置信度分级规则 (UR-07)

**归并来源**: P3 证据等级(EV-A/B/C) + P4 版本层次 + P1/P2 补充定义

```
UR-07.1 五级统一证据等级 (UEV-1 ~ UEV-5):

| 等级 | 名称 | 置信度范围 | 通用判定标志 | Part对照映射 |
|------|------|-----------|-------------|-------------|
| UEV-1 | 正式审批/法定文件 | 0.93~1.00 | 文件编号+审批标志+"经报XX批准"/"批复" | P1:含文号的名单=0.95; P3A:EV-A1=0.95; P4A:BUD-T1/T2决算=0.98 |
| UEV-2 | 官方主动公开(原始) | 0.85~0.92 | 出资人机构官网原始发布+正式栏目+无转载标志 | P1:LIST-A完整=0.90; P2:RUN-A=0.90; P3B:EV-B2=0.95; P4B:RPT-C全文=0.95 |
| UEV-3 | 新闻通稿/企业公告 | 0.65~0.84 | 政府/企业新闻通稿+有具体数据但无文号 | P2:RUN-B=0.75; P3A:EV-A3=0.80; P3C:MC-L3=0.75; P4A:报告摘要=0.75 |
| UEV-4 | 间接/审议/不完整源 | 0.45~0.64 | 审议意见/转载/部分数据/推断 | P1:LIST-B推断=0.55; P3A:EV-A4=0.60; P3C:MC-L4=0.40; P4B:RPT-R=0.60 |
| UEV-5 | 媒体/二手/无出处 | 0.20~0.44 | 财经媒体+"据悉"+无正式来源 | P2:RUN-E=0.30; P3A:EV-A5=0.40; P4B:RPT-N=0.40 |

UR-07.2 置信度计算通用公式:
  confidence = confidence_base(由UEV等级决定)
    + adjustment_data_completeness (完整数据+0.05; 仅同比-0.05; 仅数量-0.10)
    + adjustment_timeliness (3年内+0; 3~5年-0.05; >5年-0.10)
    + adjustment_cross_validation (有交叉验证+0.05; 与其他来源矛盾-0.10)
    + adjustment_format (结构化+0.03; OCR/推断-0.05)
  → 最终 confidence = max(0, min(1, 计算值))

UR-07.3 置信度输出格式（全Part统一）:
  每条输出记录必须包含:
  {
    "confidence": 0.XX,
    "evidence_level": "UEV-N",
    "confidence_factors": ["+正式发布", "+完整数据", "-过期3年"],
    "confidence_note": "一句话说明"
  }
```

### §12.3 scripts/ 脚本职责清单

> 以下仅描述脚本职责，不写代码。每个脚本标注输入/输出/风险点。

| # | 脚本名 | 职责 | 输入 | 输出 | 风险点 |
|---|--------|------|------|------|--------|
| S-01 | `url_normalizer` | URL规范化：去除tracking参数、统一协议、处理重定向、提取域名机构类型 | 原始URL字符串 | 规范化URL + source_type_code推断 | 部分政府网站有动态URL(含session/token)会被错误截断 |
| S-02 | `period_parser` | 从标题和正文提取数据期间：匹配12个正则(UR-03)、判定累计/单月/年度、输出结构化period对象 | 标题字符串+正文字符串 | `{period_type, year, start_month, end_month, is_point_in_time}` | 同一文中含多个期间需拆分；"上半年"等模糊词需映射 |

【8个脚本的设计要点（请逐一编写 spec）】

**S-01 url_normalizer.spec.md**
- 输入: 原始URL(string)
- 核心逻辑: HTTP→HTTPS标准化; 去除session/token参数; 域名→机构类型映射(S1~S8); 识别PDF/Word/Excel附件
- 关键依赖: references/source-identification-rules.md §UR-01
- 风险: 动态URL含session参数被截断后内容不同; 同一机构多域名

**S-02 period_parser.spec.md**
- 输入: 标题(string) + 正文前500字(string)
- 输出: period_type(月/季/年/累计) + year + month_start + month_end + is_cumulative + ambiguity_flag
- 核心逻辑: 应用 references/period-rules.md §R6 全部12个正则; 消歧优先级（标题>正文>URL日期）
- 风险: "1-12月"识别为累计还是年度; 多期间同文需拆分

**S-03 attachment_extractor.spec.md**
- 输入: 页面URL(string)
- 输出: attachments[](URL, type, size, is_table, is_scan, ocr_needed)
- 核心逻辑: 检测.pdf/.doc/.docx/.xls/.xlsx后缀; 扫描件PDF检测(文本层为空); 附件优先级判定(L1>L2>L3)
- 风险: 附件为扫描件需OCR; 附件URL需登录验证（政务系统常见）

**S-04 deduplicator.spec.md**
- 输入: records[](含URL、标题、part_id、entity字段)
- 输出: deduped_records[](含dup_group_id, kept_version_reason)
- 核心逻辑: 三级去重（URL完全匹配→标题+period→entity+part+period）; 取最高置信度版本
- 风险: 企业名称变体（"×× 集团" vs "××集团有限公司"）导致漏去重
- 关键依赖: references/field-dictionary.md §UR-06

**S-05 field_standardizer.spec.md**
- 输入: raw_record(dict) + part_id(P1/P2/P3A/P3B/P3C/P4A/P4B)
- 输出: standardized_record(dict) + warnings[](未能标准化的字段)
- 核心逻辑: 指标名映射（"营业收入"→"营业总收入"等13条）; 单位统一（亿/万元→元）; 百分比vs百分点区分
- 风险: 省份特异性指标名（部分地方独有表达）; "减亏"场景下负利润+正同比

**S-06 confidence_calculator.spec.md**
- 输入: record(dict含source_type_code, completeness%, period_age_days, cross_verified)
- 输出: uev_score(1-5) + confidence_float(0.0-1.0) + factors[](调整项说明)
- 核心逻辑: 基础分（按UEV等级）; -0.1/+0.1调整项（完整性<60%/-; 时效>180天/-; 交叉验证通过/+）
- 风险: 权重基于≤15个采样，可能有偏差，需定期重新校准

**S-07 keyword_matcher.spec.md**
- 输入: text(string) + part_context[](当前任务涉及的Part列表) + match_level("title"/"body"/"attachment")
- 输出: hits[](keyword, weight, part_affinity[], is_exclude_context)
- 核心逻辑: 加载 references/keyword-lexicon.md; 过滤 context_parts∩part_context=∅的词; 对C-05/C-06冲突词根据part_context差异化权重
- 风险: "预算执行"在P2中排除, 在P4A中为核心词（C-05冲突）

**S-08 balance_validator.spec.md**
- 输入: budget_record(含收入总额, 支出总额, 调出资金, 上年结余)
- 输出: is_balanced(bool) + gap(float) + balance_formula_used(string)
- 核心逻辑: 收入总额 + 上年结余 = 支出总额 + 调出资金 + 本年结余（允许±1%误差）
- 风险: 部分地方不公开结转项导致验证无法完成; 单位不统一（万元vs亿元）导致误报

【验证步骤】
完成所有8个spec后:
□ 每个spec是否标注了 MVP必须/v2扩展/可选
□ 每个spec是否引用了对应的 references/ 规则文件
□ 脚本间是否有循环依赖（例如: S-05依赖S-07的输出，而S-07依赖S-05的标准化结果）
□ 风险点是否覆盖了 failure_modes.md 中标注"需脚本处理"的项
```

**常见失败点**:
1. 写了实现代码而不是接口 spec → 超出任务范围，产生错误依赖
2. 未标注 MVP/v2 优先级 → 影响开发排期决策
3. S-07 未处理 C-05/C-06 跨 Part 关键词冲突 → 运行时误检高发
4. S-04 去重键设计过紧（仅URL匹配）→ 漏去重 URL 不同但内容相同的情形

### PROMPT-05: 一致性校验

**提示词名称**: `validate-consistency`
**使用时机**: 任何一个 skill 文件编写完成后，或批量编写后的质检阶段

**适用场景**:
- 校验 SKILL.md 与 references/ 的引用关系是否对应
- 校验字段定义与示例是否一致
- 校验关键词库与失败模式中的高频词是否对应
- 检测冲突规则残留

**输入**:
- 待校验的 skill 文件（一个或多个）
- 已知冲突列表（来自 skill_materials.md §11.0 冲突登记表 C-01~C-07）

---

**提示词正文（可直接投喂）**:

```
你是一个技术审查专家。任务是对"地方国资信息采集与分析" skill 进行一致性校验，输出校验报告，不修改文件。

【已知冲突清单（必须逐一检查）】
C-01: caliber字段在P1/P2/P4语义不同 → 是否已添加caliber_domain鉴别器
C-02: P3用announce_date，其余用publish_date → 是否已统一为publish_date
C-03: source_type_code(S1~S8)与source_institution(字符串)共存 → 是否两者均为必填
C-04: P1/P2缺confidence字段 → 是否已补充为必填
C-05: "预算执行"在P2中为排除词，在P4A中为核心词 → keyword_lexicon.md中是否按context_parts[]正确标注
C-06: "审计报告"在P2中为排除词，在P4B中为补充源 → 同C-05处理方式
C-07: 证据等级EV-A/B/C三套分立 → 是否已统一为UEV-1~5五级体系

【校验维度 1: 引用完整性】
检查 SKILL.md 中所有"→ 详见 references/xxx.md §N"的外链:
- 目标文件是否存在
- 目标章节编号是否存在（不能是占位文件）
- 如果目标为占位，标记为 [BROKEN_LINK]

【校验维度 2: 字段一致性】
抽查以下字段在所有文件中的定义:
- publish_date: 在 field-dictionary.md、examples.md 中定义是否相同
- confidence: 在 field-dictionary.md、confidence-rules.md 中公式是否一致
- caliber / caliber_domain: 在 P1/P2/P4 的字段表中是否分开定义
- source_type_code: 在 source-identification-rules.md 和 field-dictionary.md 中枚举值是否一致

【校验维度 3: 关键词-失败模式对应】
读取 keyword-lexicon.md 中权重为3的关键词（最高权重）。
检查 failure-modes.md 中是否存在对应的"漏检"或"误检"模式:
- 若高权重词在 failure-modes.md 中没有对应的失败模式 → 标记为 [COVERAGE_GAP]

【校验维度 4: 示例合规性】
检查 examples.md 中每个示例:
- 必须包含 confidence、source_type_code、publish_date、caliber_domain 四个字段
- 置信度（UEV分级 + float值）是否与所引用的证据等级一致
- period 格式是否符合 period-rules.md §R6 的输出规格

【校验维度 5: 口径警示完整性】
在 SKILL.md 中查找6条口径警示:
1. "监管企业名单" ≠ "全部国有企业名单"
2. "本级预算" ≠ "全辖区预算"
3. "预算数" ≠ "决算数"
4. "利润总额" ≠ "净利润"
5. "审议意见" ≠ "报告全文"
6. "转载新闻" ≠ "原始发布"
如果任何一条缺失或被修改 → 标记为 [CRITICAL_MISSING]

【输出格式】
```
## 一致性校验报告
检查时间: [时间]
文件列表: [列出检查的文件]

### 已知冲突检查 (C-01~C-07)
| 冲突ID | 状态 | 位置 | 说明 |
|--------|------|------|------|
| C-01   | ✓已解决 / ✗未解决 / ？无法确认 | ... | ... |

### 引用完整性
[BROKEN_LINK] 列表（如无则写"全部引用有效"）

### 字段一致性
[INCONSISTENCY] 列表（如无则写"字段定义一致"）

### 关键词覆盖
[COVERAGE_GAP] 列表（如无则写"高权重词均有对应失败模式"）

### 示例合规性
[NON_COMPLIANT] 列表（如无则写"所有示例合规"）

### 口径警示
[CRITICAL_MISSING] 列表（如无则写"6条警示完整"）

### 总体结论
[PASS / FAIL / WARN]
阻断项: [列出 CRITICAL 级问题]
警告项: [列出 WARN 级问题]
```

【重要】仅输出校验报告，不修改任何文件。修改工作由另一任务执行。
```

**预期输出**:
- 结构化校验报告（通过/失败/警告三级）
- 列出所有阻断项（CRITICAL）和警告项（WARN）
- 不修改任何文件

**常见失败点**:
1. 校验时自动"顺手修复"问题 → 混淆校验和修复两个阶段，版本管理混乱
2. 仅检查 SKILL.md 而不检查 references/ → 遗漏大量不一致
3. 未检查 C-05/C-06 关键词跨 Part 冲突 → 运行时误检不被发现
4. 对占位文件给出虚假"引用有效" → 掩盖 BROKEN_LINK

### PROMPT-06: 版本迭代

**提示词名称**: `iterate-skill-version`
**使用时机**: MVP 已验证，需要升级到 v2；或 v2 运行后发现规则缺陷，需要局部修订

**适用场景**:
- MVP → v2 全量升级
- 单 Part 规则修订（如新发现的失败模式）
- 关键词库迭代（新省份采样后更新）
- 字段扩展（新增指标）

**输入**:
- 当前版本文件
- 升级触发事件（省份验证发现问题 / 新采样数据 / 用户反馈）
- 升级范围（全量 / 指定文件 / 指定规则）

---

**提示词正文（可直接投喂，填写 [TRIGGER] 和 [SCOPE]）**:

```
你是一个 skill 版本管理专家。任务是基于触发事件，对"地方国资信息采集与分析" skill 进行受控迭代。

【触发事件】: [TRIGGER]
（例: "浙江省端到端验证发现 Part 1 关键词漏检率 23%，主要原因是缺少'省属企业'作为P1关键词"）

【升级范围】: [SCOPE]
（例: "仅更新 references/keyword-lexicon.md 的 P1 部分，同步更新 SKILL.md §3 P1决策树"）

【版本迭代规则（必须遵守）】

规则 V-01: 任何修改前，先在文件头更新版本号和日期
  格式: `版本: v[N].[M] | 更新: YYYY-MM-DD | 变更: [一句话摘要]`

规则 V-02: 修改关键词时
  - 新增词: 标注 `+v[N].[M]`，说明来源（哪个省份/案例触发）
  - 删除词: 不直接删除，改为 weight=0 并标注 `deprecated v[N].[M]`
  - 调整权重: 原权重值保留为注释

规则 V-03: 修改字段定义时
  - 若修改影响已有示例 → 同步更新 references/examples.md 对应示例
  - 若新增字段 → 检查 field_standardizer.spec.md (S-05) 是否需要同步

规则 V-04: 修改失败模式时
  - 新增 FM 编号: 在现有最大编号后顺序添加，不复用已废弃编号
  - 废弃 FM: 标注 `DEPRECATED v[N].[M]`，原因说明

规则 V-05: 修改后必须执行一致性校验
  使用 PROMPT-05 (validate-consistency) 对修改涉及的文件进行校验
  校验通过（无 CRITICAL 项）才能提交版本

规则 V-06: 不允许的修改（需升级协议）
  - 修改6条口径警示的内容 → 需重新走 PROMPT-02 完整流程
  - 修改 UEV 等级映射 → 需同步更新全部 confidence 示例（R-09）
  - 删除 Part 或合并 Part → 需重新走 PROMPT-01+02+03 完整流程

【升级流程（按触发类型选择）】

类型 A: 关键词迭代（新省份采样）
  1. 读取新采样数据，提取新命中词和新误检词
  2. 更新 references/keyword-lexicon.md（遵循V-02）
  3. 检查是否影响 S-07 keyword_matcher.spec.md
  4. 执行 PROMPT-05 校验维度3（关键词覆盖）
  5. 更新版本号

类型 B: 失败模式新增
  1. 描述新失败场景: 触发条件 + 影响 Part + 预防方式
  2. 按 failure_modes.md 格式添加新 FM 条目
  3. 检查是否需要同步修改 SKILL.md guardrails
  4. 执行 PROMPT-05 校验维度3
  5. 更新版本号

类型 C: 字段扩展
  1. 在 references/field-dictionary.md 新增字段条目
  2. 在 references/examples.md 至少一个示例中添加该字段
  3. 在 scripts/field_standardizer.spec.md (S-05) 更新处理逻辑说明
  4. 执行 PROMPT-05 校验维度2
  5. 更新版本号

类型 D: MVP → v2 全量升级
  1. 读取 skill_materials.md §12.5 的 v2 定义（18个 v2 新增文件/内容）
  2. 按优先级补充: R-04 → R-07 → R-08 → R-09(10例) → S-04~S-08 → A-02~05
  3. 每补充一个文件，立即执行 PROMPT-05 对该文件校验
  4. 全部补充完成后执行全量校验
  5. 升级版本号为 v2.0

【输出要求】
完成迭代后输出:
```
## 迭代报告
版本: [旧版本] → [新版本]
触发: [TRIGGER摘要]
变更文件: [列出所有修改的文件]
变更摘要:
  新增: [列出新增内容]
  修改: [列出修改内容]
  废弃: [列出废弃内容]
校验结果: [PASS/WARN/FAIL]
下一步建议: [若校验有WARN项，列出建议处理方式]
```
```

**预期输出**:
- 受控修改（带版本号和变更记录）
- 迭代报告（含变更摘要和校验结果）
- 不改变未受影响的文件

**常见失败点**:
1. 直接修改内容而不更新版本号 → 失去变更可追溯性
2. 修改关键词时直接删除条目 → 下次迭代无法判断"是否考虑过这个词"（应标 deprecated）
3. 字段扩展后未更新示例 → 校验维度2 报 INCONSISTENCY
4. MVP→v2 升级时一次性提交所有文件 → 出错时无法定位问题；应逐文件提交并校验

### §14.1 PATCH-01: R4 名单口径规则补充

```
R4.4 省份差异化口径判断:
  广东: 国资委监管企业含金融类(粤科金融集团), includes_financial=true
  多数省份: 金融类国企由金融监管局/财政厅监管, 不在国资委名单中
  IF province="广东" AND list_caliber="supervised" → includes_financial=true
  IF province IN (大多数) AND list_caliber="supervised" → includes_financial=false

  IF source_page_type="LIST-F" (首页链接):
    检查项: 链接总数是否合理(通常10~40个集团)
    检查项: 链接是否含企业官网URL(指向独立域名=高置信)
    检查项: 是否有"查看更多"或分页按钮(有→可能不完整)
    检查项: 与同省其他来源核对数量(如年报中提及的户数)

R4.6 identifiable口径置信度映射:
  caliber=identifiable 时置信度上限为 0.65:

R4.7 无名单省份处理规则(LIST-B模式):

    Step 4: 标注 completeness="partial_identified" + note="无独立名单页(LIST-B)"
```

### §14.3 PATCH-03: R6 期间识别规则补充

```
R6.5 同文多期间拆分规则:
  IF 同一篇文章含单月数据和累计数据(如浙江同时报告1月和1-2月):
    拆为两条独立记录, 分别标注:

R6.7 时点数据期间表示:

### §14.4 PATCH-04: R7 指标标准化规则补充

```
R7.5 调整后同比处理:
  "剔除政策性因素后同比增长X%" →
    生成独立记录: indicator_name="{原指标}(剔除政策性因素后同比)"
    yoy_pct=X, yoy_adjusted=true

  "应交税金" → "应交税费" (旧称)
  "总资产" → "资产总额"
  "省属企业" → 口径B1(省属监管)的同义表达
  "财务快报" → is_preliminary=true

### PATCH-13: 预算数字/制度规则拆分规则 (BDG-01~03)

**BDG-01 record_type二分**
- 每条预算相关记录必须标注 `record_type` ∈ {BUDGET_FIGURE, RULE, TRANSFER}
  - BUDGET_FIGURE: 某级预算的收入/支出/调入金额
  - RULE: 净利润上缴比例、分类分档、转入一般公共预算比例等制度性规定
  - TRANSFER: 上下级间转移支付金额
- 判定逻辑: 含"比例""百分比""收取比例""上缴比例""分为X类" → RULE; 含"预算收入""预算支出""亿元" → BUDGET_FIGURE; 含"转移支付""对地方" → TRANSFER

**BDG-02 比例规则来源层次**
- 中央级净利润上缴比例: 来源="关于20XX年中央国有资本经营预算的说明"(财政部预算司)
- 省级净利润上缴比例: 来源="XX省国有资本收益收取管理办法"(省财政厅/省政府)
  - 注意: 省级比例**不在**预算草案报告中, 需**单独搜索**省级办法类文件
- 如在预算草案中未找到比例, 必须搜索对应层级的"管理办法"类文件
- 如仍未找到, 标注 ratio_value = "UNIDENTIFIED"

**BDG-03 分类分档完整性校验**
- 中央4类比例: 必须确认全部4类的比例值和适用企业范围
  - 如某类被截断/不可读, 标注该类 evidence_strength = S3, ratio_value = "UNIDENTIFIED"
  - 不得推断: 历史值≠当前值, 除非有当年文件确认
- 省级分类分档: 各省档数和比例值可能与中央不同, 不可套用中央标准

### PATCH-14: 本级/全辖区判定规则 (SCOPE-01~04)

**SCOPE-01 scope多级编码**
- scope字段必须使用以下编码:
  - `national` = 全国合计(中央+全部地方)
  - `central` = 中央级(中央企业)
  - `local_aggregate` = 全国地方合计(不分省)
  - `province` = 省级
  - `city` = 地级市
- scope_label补充说明: `central_budget`, `central_soe`, `national_total`, `local_aggregate`, `province_level`, `central_to_local`

**SCOPE-02 "本级"消歧**
- "地方国有资本经营预算**本级**收入" 在全国预算草案中 → scope = `local_aggregate` (不是某省本级)
- "XX省**本级**国有资本经营预算" 在省预算草案中 → scope = `province`, scope_label = `province_level`
- "XX市**本级**" → scope = `city`
- 消歧键: 文件来源. 全国两会报告 → local_aggregate; 省人大报告 → province

**SCOPE-03 转移支付≠本级预算**
- "中央对XX省国有资本经营转移支付" (如广东0.75亿元) → record_type = TRANSFER
  - 该金额**不是**XX省国有资本经营预算总量, 仅为中央→地方划拨的一小部分
  - 不得将此金额用于表示该省预算规模
- 各省本级预算总量需从省级预算草案获取

**SCOPE-04 全省统计描述标注**
- 如果某省报告中出现"全省国有资本经营预算收入", 需判断:
  - 是否含省本级+下辖市县? 如是, scope = `province`, scope_label = `province_total` (非本级)
  - 是否仅省本级? scope = `province`, scope_label = `province_level`
- 判定依据: 标题含"汇总"/"全省" → province_total; 标题含"本级"/"省级" → province_level

### PATCH-15: 特殊规定识别规则 (SPEC-01~03)

**SPEC-01 特殊预算条目清单**
- 以下条目需特殊标注:
  - 新疆生产建设兵团体制性收入/支出 → 不计入中央, 已计入地方
  - 上年结转收入 → 非当年新增, 收入总量=当年收入+结转

  - 产权转让收入/清算收入 → 一次性收入, 不可预测
- 标注字段: notes 中注明特殊性质

- 中央: 收入总量(3976.23) = 当年收入(3716.32) + 上年结转(259.91)
- "收入预算数" → 当年收入; "收入总量" → 含结转
- 对比或排序时应使用"当年收入", 不含结转

**SPEC-03 支出结构标准化**
- 中央支出4大项: 补充社保基金 / 解决历史遗留问题及改革成本 / 国有企业资本金注入 / 国有企业公益性补贴
- 还有: 调入一般公共预算(不算支出, 算调出)

## §17 网站展示层规则补丁（模块E实战追加）

> 来源: 模块E网站构建实战复盘
> 问题编号: E5-01 ~ E5-08
> 新增规则: 16条 (PATCH-18 ~ PATCH-22)

### PATCH-18: 跨交付物字段统一规则 (UNIFY-01~04)

**UNIFY-01 证据回链字段统一命名**
- 所有模块在输出CSV/JSON时, 证据回链必须包含以下4个标准字段:
  - `evidence_url`: 来源URL（不论底表原始字段名为source_url还是evidence_url）
  - `evidence_strength`: 证据等级（统一为string: S1/S2/S3/S4, 不论底表用int还是string）
  - `evidence_institution`: 来源机构（不论底表原始字段名为source_institution还是issuer）
  - `evidence_date`: 来源日期（不论底表用source_date/publish_date/extraction_date）
- 底表可保留原始字段名, 但在传入前端接口前, 必须映射为上述标准名

**UNIFY-02 省份编码统一标准**
- 省份编码统一使用两位大写字母: GD/ZJ/SD/SC/BJ/SH/JS/HB...
- 全国使用 CN, 兵团使用 BT
- 底表可同时保留全拼(guangdong)或中文(广东), 但必须有province_code标准字段
- 跨模块JOIN时, 以province_code为唯一连接键

**UNIFY-03 record_type枚举统一**
- 所有包含多种记录类型的CSV, record_type字段必须使用以下枚举值之一:
  - ENTITY / DATA / RULE / BUDGET_FIGURE / TRANSFER / POLICY / UNIDENTIFIED
- 前端根据record_type决定路由至对应栏目

**UNIFY-04 主键字段规范**
- 每个CSV必须有唯一主键字段, 命名规范: `{module_prefix}_id`
- 模块A: entity_id (E-GD-0001)
- 模块B: record_id (MO-CN-202601-02-rev)
- 模块C: policy_id (POL-001)
- 模块D: budget_id (BDG-R-001)
- 前端通过主键实现行级证据展开和跨模块关联

### PATCH-19: 前端接口模板规则 (API-01~03)

**API-01 数据接口规范**
- 所有数据通过CSV文件以静态方式提供, 路径: data/{module_name}.csv
- 前端通过fetch()加载后在客户端解析
- 接口清单:

| 接口ID | 路径 | 底表 | 列数 | 主键 | 供给栏目 |
|--------|------|------|------|------|---------|
| API-A | data/entity_list.csv | 模块A | 21 | entity_id | 栏目1企业主体 |
| API-B | data/monthly_operation.csv | 模块B | 28 | record_id | 栏目2经营数据 |
| API-C | data/policy_briefing.csv | 模块C | 15 | id | 栏目3收益规则(政策) |
| API-D1 | data/budget_rules.csv | 模块D | 21 | record_id | 栏目3(RULE)+栏目4(BUDGET_FIGURE/TRANSFER) |
| API-D2 | data/budget_transfer.csv | 模块D | 3 | province | 栏目4转移支付明细 |

**API-02 单表多栏目拆分规则**
- 当一个CSV的record_type包含多种类型时, 按以下规则拆分至不同栏目:
  - record_type=RULE → 栏目3(收益管理规则)
  - record_type=BUDGET_FIGURE → 栏目4(财政统筹结果)
  - record_type=TRANSFER → 栏目4(财政统筹结果)
- 前端应在数据加载后通过filter()拆分, 不需要物理拆分CSV

**API-03 筛选维度标准**
- 每个栏目必须提供以下筛选维度:
  - 栏目1: province(省份), list_caliber(口径)
  - 栏目2: scope_code(口径), indicator_name(指标), province(省份)
  - 栏目3: doc_type_code(文种等级), topic_primary(主题), ratio_type(比例类型)
  - 栏目4: scope(范围), record_type(记录类型)

### PATCH-20: 证据回链机制规则 (EVLINK-01~03)

**EVLINK-01 证据面板标准结构**
- 每条数据行可展开证据面板, 必须包含以下4项:
  1. 来源机构 (evidence_institution)
  2. 来源文件链接 (evidence_url → 可点击跳转至原始页面)
  3. 证据等级badge (evidence_strength → S1绿/S2黄/S3橙/S4红)
  4. 采集日期 (evidence_date)
- URL链接必须经过协议校验, 仅允许http/https

**EVLINK-02 证据回链跨模块适配**
- 各模块底表字段名不同时, 前端需通过适配器映射:

| 底表字段 | 标准字段 | 适用模块 |
|---------|---------|---------|
| source_url | evidence_url | A, B, D |
| evidence_url | evidence_url | C |
| uev_level (int) | evidence_strength (S{n}) | A, B |
| evidence_strength (str) | evidence_strength | C, D |
| source_institution | evidence_institution | A, B |
| issuer | evidence_institution | C |
| source_doc | evidence_doc | D |
| source_title | evidence_doc | B |
| title | evidence_doc | C |

**EVLINK-03 UNIDENTIFIED证据处理**
- 当evidence_url为空且数据值为UNIDENTIFIED时:
  - 不渲染链接, 显示"—"
  - evidence_strength自动标注为S4
  - 证据面板显示"未在公开材料中识别"

### PATCH-21: 图表组件约束规则 (CHART-01~03)

**CHART-01 图表最小字段集**
- 柱状图(对比): 至少需要 indicator_name + value + year (2个以上对比组)
- 横条图(排名): 至少需要 dimension_label(如province) + metric_value, 数据≥3条
- 折线图(趋势): 至少需要 time_axis(year/month) + value, 时间点≥3个
- 如不满足最小字段集, 不渲染图表, 显示"数据不足暂无法绘制图表"

**CHART-02 空态/稀疏数据处理**
- 当筛选后结果为0条: 显示"暂无符合条件的数据"(灰色文字+图标)
- 当结果仅1条: 显示数值卡片而非图表
- 当缺少对比维度(如仅1个省份): 降级为表格展示, 不强制绘图

**CHART-03 图表字段与底表对齐验证**
- 图表渲染前校验: 所引用的字段(如indicator_name, value, province)必须存在于底表中
- 不允许使用底表中不存在的生造字段
- 图表下方标注数据来源: "数据来源: {csv文件名}, 字段: {field1}×{field2}"

### PATCH-22: 简报附录映射规则 (BRIEF-01~03)

**BRIEF-01 政策简报内容分层展示**
- 政策简报在网站端分3层展示:
  1. 列表层: 仅显示title, issuer, doc_date, doc_type, topic_primary (来自CSV)
  2. 摘要层: 展开显示summary字段内容 (来自CSV)
  3. 全文层: 通过evidence_url链接至原始来源页面
- Word简报(.docx)中的章节结构不在网站端还原, 仅通过下载链接提供

**BRIEF-02 约束性表述在线展示**
- policy_briefing.csv的binding_phrases字段应在展开详情时显示
- 格式: 逐条列出约束短语, 按约束级别标色(应当=蓝, 不得=红, 鼓励=绿)

**BRIEF-03 财政统筹与预算规则栏目映射**
- 栏目4(财政统筹)必须与模块D budget_rules.csv完全对齐:
  - BUDGET_FIGURE类: 在"预算收支概览"表格中展示
  - TRANSFER类: 在"转移支付明细"表格中展示
  - RULE类: **不在栏目4展示** → 路由至栏目3
- 栏目3(收益规则)展示policy_briefing.csv全部 + budget_rules.csv中record_type=RULE的记录

## §18 官方链接注册表补丁

### PATCH-23: 地方国资官网链接注册表规则 (SRC-REG-01~06)

**SRC-REG-01 官方链接注册表优先**
- 对省级/直辖市/兵团国资委的任何采集任务，第一步必须读取注册链接表。
- 本项目的注册链接表文件为:
  - `delivery/sasac_official_link_registry.md`
  - `delivery/sasac_website_registry.csv`
- 不允许在未查表的情况下直接凭经验猜测域名。

**SRC-REG-02 中央国资委列表为权威上游**
- 地方国资委官网链接的权威上游页面固定为:
  - `http://www.sasac.gov.cn/n4422011/n17627531/c17633273/content.html`
- 如果某省 URL 访问失败，必须先回该页面核对锚点链接，再决定是否更新注册链接表。

**SRC-REG-03 官网优先于新闻与转载**
- 国企名单、收益管理规则、监管制度、政策原文的最终来源必须是对应省级国资委官网及其同域名栏目页。
- 国务院国资委“地方”栏目、省政府新闻频道、门户新闻稿、企业新闻稿、第三方转载页只能作为线索，不能替代省级国资委官网来源。

**SRC-REG-04 失败链接处理顺序**
- Step 1: 读取注册链接表中的 `sasac_url`
- Step 2: 访问官网
- Step 3: 若失败，回中央国资委地方国资委页面核对链接
- Step 4: 更新注册链接表中的 `sasac_url` / `enterprise_list_url` / `status` / `notes`
- Step 5: 仅从确认后的官网域名继续抽取名单或政策

**SRC-REG-05 反爬阻断不得替换来源**
- 若正确官网 URL 已确认，但当前环境遇到 WAF、JS challenge、Cookie challenge、403、412 或 502:
  - 保留官方 URL
  - 在注册链接表中标记 `BLOCKED` 或 `PARTIAL`
  - 在 `notes` 中记录阻断方式与测试日期
  - 不得用新闻页或转载页改写 `sasac_url`

**SRC-REG-06 省级信息源路由修正**
- 对 Part1 名单、Part3 政策、Part4 收益管理规则的省级采集，新增强制前置检查:
  - 先定位 `sasac_url`
  - 再定位 `enterprise_list_url` 或政策栏目页
  - 未完成这一步，不进入正文抽取
